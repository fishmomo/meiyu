# CRA40 按日期目录归类与兼容迁移设计

## 目标

将当前散落在 `data/raw/cra40/` 根目录的 2017 年 6 月 CRA40 原始资料，迁移到与现有 2017 年 5 月冷涡资料一致的年/日期目录结构，同时保持现有梅雨新流程与旧版脚本可读取这些资料。

## 目标目录

```text
data/raw/cra40/
├─ 2017/
│  ├─ 20170501/
│  │  ├─ CRA40_GPH_*_GLB_2P50_*.grib2
│  │  └─ CRA40_TEM_*_GLB_2P50_*.grib2
│  └─ 20170621/
│     ├─ CRA40_RHU_*_GLB_0P25_*.grib2
│     ├─ CRA40_TEM_*_GLB_0P25_*.grib2
│     ├─ CRA40_VVP_*_GLB_0P25_*.grib2
│     ├─ CRA40LAND_PRECIP_*.grib
│     ├─ CRA40_*.nc
│     └─ 对应的 `.idx` 文件
```

文件以文件名中的十位时间标记 `YYYYMMDDHH` 决定目标日期目录；`.idx` 文件随同其对应原始 GRIB 文件迁移。

## 兼容读取

`project_paths.cra40_file(filename)` 的新规则：

1. 优先保留根目录文件的读取兼容；
2. 根目录没有目标文件时，在 `data/raw/cra40/<year>/<YYYYMMDD>/` 查找；
3. 找不到或发现多个同名候选时，报出明确错误，不静默选择不确定文件。

`cra40_glob(pattern)` 也需覆盖年/日期子目录，以保持旧版 `frontal_processing_CRA40.py` 对 `CRA40*.nc` 的批量读取能力。

## 范围

本次仅整理已存在的 2017 年 6 月 CRA40 原始资料；不改动梅雨个例的 manifest、锋面掩膜、诊断算法或输出目录。五月冷涡资料维持现状。

## 验证

在迁移前后分别验证：

1. 日期目录解析与根目录兼容的单元测试；
2. `cra40_file()` 能解析既有梅雨 RH/T/W 文件；
3. `cra40_glob("CRA40*.nc")` 能找回全部旧版综合场；
4. 已有 CRA40 manifest 的输入解析与相关测试通过。

## 风险控制

文件移动只在兼容定位逻辑和测试通过后执行。迁移脚本应先做预演清单，确认每个源文件与目标文件一一对应、无重名冲突，再执行实际移动。
